Web Uygulamaları Hosting Altyapısı

Hazırlayan: Oğuzhan İnandı  |  Tarih: 20.04.2026  |  Versiyon: 1.0

1. Genel Bakış

Bu dokümanda, mevcut Vercel platformunda çalışan PLM BOM Entegrasyon uygulaması ve benzer birkaç web uygulamasının şirket içi sanal sunucuya taşınması için gereken sunucu gereksinimleri detaylandırılmaktadır.

ParametreDeğer
Mevcut Uygulama Sayısı1 (PLM BOM Entegrasyon) + planlanmış 2-3 ek uygulama
Beklenen Kullanıcı Sayısı10-50 eşzamanlı kullanıcı (şirket içi)
Erişim TipiŞirket içi ağ (intranet) — VPN ile uzaktan erişim opsiyonel
Çalışma Saatleri7/24 (kesintisiz)

2. Donanım Gereksinimleri

Kaynak Minimum Önerilen (Birden Fazla Uygulama İçin)
İşlemci (vCPU) 4 vCPU 8 vCPU (x86_64)
RAM 8 GB 16 GB
Disk (SSD) 80 GB SSD 200 GB SSD (NVMe tercih edilir)
Ağ 1 Gbps 1 Gbps (şirket içi LAN)
Not: Disk alanı hesabı: İşletim sistemi ~10 GB, Docker images ~15 GB, PostgreSQL verileri ~20 GB, yüklenen Excel dosyaları ~10 GB, log dosyaları ~5 GB, yedekleme alanı ~30 GB + gelecek uygulamalar için boşluk.

5. Ağ & Port Gereksinimleri

PortProtokolServisYön
80TCP (HTTP)Nginx (→ HTTPS redirect)Inbound
443TCP (HTTPS)Nginx reverse proxyInbound
22TCP (SSH)Sunucu yönetimiInbound
3000TCPNext.js (internal)Internal only
8000-8005TCPBackend API (internal)Internal only
5432TCPPostgreSQL (internal)Internal only
443TCPGitHub (git pull)Outbound
443TCPnpm/pip registryOutbound
Önemli: Port 3000, 8000-8005 ve 5432 sadece sunucu içi (localhost/Docker network) erişimde olacak, dışarıya sadece 80/443 (Nginx üzerinden) ve 22 (SSH) açık olmalıdır.

7. Domain & DNS Gereksinimleri

ParametreDeğer
Hostname (örnek)apps.temsa.local veya entegrasyon.temsa.com.tr
DNS KaydıA kaydı → sunucu IP adresi (şirket içi DNS sunucusuna)
Ek UygulamalarSubdomain veya path-based routing (Nginx ile)
Örnek yapı: entegrasyon.temsa.local → PLM BOM Entegrasyon  |  app2.temsa.local → Diğer uygulama  |  app3.temsa.local → Diğer uygulama

9. İzleme & Bakım

  • Disk doluluk izleme — %85 üzerinde uyarı
  • Bellek/CPU izleme — aşırı kullanımda uyarı
  • Docker container sağlık kontrolü — otomatik restart
  • Log rotasyonu — logrotate ile haftalık rotasyon
  • Uptime izleme — HTTP health-check endpoint
